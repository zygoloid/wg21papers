Core Issue 1839: Lookup of block-scope extern declarations

Change in 3.5 basic.link paragraph 6:

The name of a function declared in block scope and the name of a variable declared by a block scope extern declaration have linkage. A prior declaration is looked up (3.4.1 [basic.lookup.unqual]) in the innermost enclosing namespace scope. If this lookup finds a there is a visible declaration of an a namespace scope entity with linkage having the same name and type, ignoring entities declared outside the innermost enclosing namespace scope, the block scope declaration declares that same entity and receives the linkage of the previous declaration. If there is more than one such matching entity, the program is ill-formed. Otherwise, if no matching entity is found, the block scope entity receives external linkage.
