Core Issue 1652: Object addresses in constexpr expressions

Change in 5.10 expr.eq paragraph 2:

If at least one of the operands is a pointer, pointer conversions (4.10) and qualification conversions (4.4) are performed on both operands to bring them to their composite pointer type (Clause 5). Comparing pointers is defined as follows:
  • If one pointer represents the address of a complete object, and another pointer represents the address one past the last element of a different complete object [Footnote: An object that is not an array element is considered to belong to a single-element array for this purpose; see 5.3.1. -- end footnote], the result of the comparison is unspecified.
  • Otherwise, Two pointers compare equal if theythe pointers are both null, both point to the same function, or both represent the same address (3.9.2), they compare equal.
  • Otherwise they, the pointers compare unequal.
